# 公立医院算法岗位情况 (v27)

> ⚠️ **重要发现：公立医院基本不公开招聘算法工程师**

---

## 核心结论

**国内公立医院（顶级三甲）几乎不发校招算法岗。** 验证过华山医院，搜"算法"/"信息"/"数据"/"AI"/"工程" 全部 0 岗位，仅有医生、护士、科研助理、临床试验专员。

---

## 为什么？

| 原因 | 说明 |
|---|---|
| **编制限制** | 公立医院属于事业单位，编制审批严格，AI 工程师一般挂在"信息中心"或"科研处"，按编外/聘任制招 |
| **业务规模** | 顶级三甲的 AI 业务通常外包给合作企业（联影/数坤/推想），医院只出数据和需求 |
| **学科门类** | 医院招聘核心是医生/护士/医技/科研，算法不属于医院"主营业务" |
| **薪资体系** | 公立医院薪资天花板低（年薪 30w 顶天），留不住 AI 人才 |

---

## 真实可去的 4 种路径

### 1️⃣ 联合培养（学制内 + 在编）

| 类型 | 例子 |
|---|---|
| **临床医学+AI 双学位** | 北京协和医学院 + 清华/北航（4+4 医工交叉） |
| **AI 硕博 + 医院实践** | 中科院计算所/自动化所/北航 + 北京协和/华西联合培养 |
| **生物医学工程** | 教育部 BME 强校（清华/上交/东南/华科/西交/北航）毕业进医院影像科/信息中心 |

**优势**：学历在高校，导师有课题，毕业可直接进医院科研岗
**劣势**：周期长（5+ 年），导师难找

### 2️⃣ 顶级三甲研究型部门（直接招）

少数顶级三甲有**正式算法岗**（事业编/聘任制），但要走医院官网公告，不是校招系统：

| 医院 | 算法相关岗位方向 | 入口 |
|---|---|---|
| **北京协和医院** | 疑难重症及罕见病国家重点实验室 / 临床生物样本库 | `pumch.cn` → 科学研究 → 人才招聘（公告制） |
| **解放军总医院(301)** | 医疗大数据中心 / 人工智能学组 | `301hospital.com.cn` → 公告 |
| **华西医院** | 医学大数据中心 / 人工智能研究室 | `wchscs.cn`（官网改版中，公告不固定） |
| **瑞金医院** | 智慧医疗 / 上海交通大学医学院附属 | `rjh.com.cn` |
| **华山医院** | 神经科学 / 影像医学 | `huashan.org.cn` (招医生为主，AI岗零星) |
| **中山医院** | 智慧医院 / 大数据中心 | `zs-hospital.sh.cn` (Cloudflare 521 防护) |
| **阜外医院** | 心血管大数据 / 影像 | `fuwai.com` |
| **宣武医院** | 神经科学 / 脑科学 | `xwhosp.com.cn` |
| **湘雅医院** | 智慧医疗 | `xiangya.com.cn` |

**特点**：公告制，无固定招聘时间。需要长期关注官网通知 + 微信公众号。

### 3️⃣ 医院 AI 企业（编制在企业，派驻医院）

这是**最常见的算法工程师进医院方式**：

| 企业 | 派驻模式 | 适合谁 |
|---|---|---|
| **联影智能** | 派驻各三甲影像科 | CV 背景 |
| **数坤科技** | 派驻心血管/卒中中心 | CV 背景 |
| **医渡科技** | 派驻医院科研处 | NLP 背景 |
| **零氪科技** | 派驻肿瘤医院 | NLP 背景 |
| **卫宁健康** | 派驻医院信息科 | 系统工程 |
| **创业慧康** | 派驻医院信息中心 | 系统工程 |

**特点**：编制在企业，工资按互联网水平发，但工作内容是医院 IT 项目。
**优势**：薪资高、不需要医学背景、能接触真实医院数据
**劣势**：编制不在医院

### 4️⃣ 国家/省级重点实验室（学术编制）

| 实验室 | 主体 | 算法方向 |
|---|---|---|
| **分子影像重点实验室** | 中科院自动化所/深圳先进院 | 医学影像AI |
| **脑科学与类脑研究中心** | 中科院神经所/北脑中心 | 脑科学+AI |
| **国家健康医疗大数据研究院** | 中国移动/浪潮/各三甲 | 医疗大数据 |
| **转化医学国家重大科技基础设施** | 上海交大+瑞金+仁济 | 转化医学+AI |
| **国家中医药数据中心** | 中国中医科学院 | 中医AI |

**特点**：有正式编制/助理研究员岗位，薪资按科研院所标准
**入口**：通过学组 PI 招收，官网单独发布

## 实战建议

### 如果你想进医院当算法工程师

| 你的情况 | 推荐路径 |
|---|---|
| 应届生，想留医学AI方向 | 路径 3（联影/数坤/医渡）最实际 |
| 在读硕博，导师是医学AI | 路径 1（联合培养/继续读博） |
| 想拿编制 + 户口 | 路径 4（国家实验室助理研究员） |
| 顶级三甲情结 | 路径 2（持续关注医院公告） |
| 想去医院信息中心 | 路径 3（卫宁/创业慧康的医院 IT 部） |

### 不要做的事

- ❌ 投"医院算法岗" - 不存在校招版
- ❌ 期望年薪百万 - 公立医院算法岗 20-30w 封顶
- ❌ 把医院当主业 - 医院 AI 业务成熟度远不如互联网/AI企业
- ❌ 放弃互联网方向 - 临床医学知识+算法的人，去互联网医疗/AI制药更吃香

## 总结

| 指标 | 数据 |
|---|---|
| 公立医院公开校招算法岗 | 0 个（华山医院实测 0） |
| 路径 1 联合培养 | 5+ 年 |
| 路径 2 顶级三甲公告 | 零星、年均 5-10 个全国 |
| 路径 3 派驻企业 | 真实就业主渠道 |
| 路径 4 国家实验室 | 学术编制 |
| 推荐指数 | 路径 3 > 路径 4 > 路径 1 > 路径 2 |

**实用结论**：**想当医院里的算法工程师，实际去 AI 医疗企业（联影/数坤/医渡等）派驻，工作内容一样，薪资高 2-3 倍**。真正"医院编制"的算法岗，凤毛麟角 + 薪资封顶。
